# 产品战略（Argus Dual-Agent System）

## 1. 产品使命

让复杂任务自动化从“单 Agent 碰运气”升级为“多 Agent 协同可控执行”。

## 2. 目标用户

- 自动化工程师：搭建企业内流程自动化
- AI 应用开发者：验证多 Agent 路由策略
- 研究与竞赛团队：快速迭代 Agent 组合能力

## 3. 核心场景

- 场景 A：桌面操作与代码执行交替的复合任务
- 场景 B：失败后自动切换 Agent 继续推进
- 场景 C：从 GUI 原型迁移到 CLI 批处理

## 4. 差异化定位

- 相比单 Agent：具备路由与兜底，成功率更稳定
- 相比重框架：结构轻量、易改造、可快速接入现有工具

## 5. 北极星指标（North Star Metric）

- 端到端任务成功率（在限定步骤与时间内完成目标任务的比例）

## 6. 关键指标

- 平均任务完成时长
- 首次成功率（不触发回退）
- 回退后恢复成功率
- 工具调用失败率
- 每任务平均 Token 消耗

## 7. 产品路线

- V1（当前）：双 Agent + Smart Router + Tool Calling
- V2：任务分解器与多阶段计划执行
- V3：可观测性面板（成功率、耗时、失败拓扑）
